# RedSquare Screens - TV Platforms

RedSquare Screens supports multiple TV and streaming device platforms, enabling you to transform any modern display into a digital advertising screen.

## Supported Platforms

### Mobile Platforms
- **[Android Mobile](./android-mobile/)** - Native Android phones and tablets
- **[iOS](./ios/)** - iPhones and iPads with casting capabilities

### Desktop Platforms  
- **[Windows](./windows/)** - Windows desktop application
- **[macOS](./macos/)** - macOS desktop application
- **[Linux](./linux/)** - Linux desktop application

### TV and Streaming Platforms
- **[Android TV](./android-tv/)** - Android TV devices and smart TVs
- **[Amazon Fire TV](./amazon-fire/)** - Fire TV Sticks, Cubes, and Edition TVs
- **[Roku](./roku/)** - All modern Roku devices and Roku TVs
- **[Samsung TV](./samsung/)** - Samsung Smart TVs (Tizen OS)
- **[LG TV](./lg-webos/)** - LG Smart TVs (webOS)

## Installation Methods

### Official App Stores (Recommended)
RedSquare Screens will be available through official app stores for easy installation and automatic updates.

### Developer/Sideloading (Current)
During the beta phase, apps can be installed through developer modes or sideloading methods:
- **Android TV**: ADB installation or APK sideloading
- **Fire TV**: ADB or Downloader app installation
- **Roku**: Developer mode installation
- **Samsung**: Tizen Studio deployment
- **LG**: webOS Developer Mode or USB installation

## Hardware Requirements

### Minimum Requirements (All Platforms)
- **Display**: 1920x1080 (Full HD) minimum
- **RAM**: 1GB minimum (2GB recommended)
- **Storage**: 500MB available space
- **Network**: 5 Mbps internet connection
- **Year**: 2018 or newer for most platforms

### Recommended Specifications
- **Display**: 4K (3840x2160) with HDR support
- **RAM**: 2GB or more for smooth performance
- **Storage**: 1GB+ for content caching
- **Network**: 25+ Mbps for optimal content delivery
- **Ethernet**: Wired connection for commercial installations

## Content Compatibility

### Supported Formats
- **Images**: JPEG, PNG, WebP, GIF
- **Videos**: MP4, WebM, MKV (platform dependent)
- **Streaming**: HLS, DASH protocols
- **Web Content**: HTML5, CSS3, JavaScript

### Content Optimization
- **Resolution**: Automatically scaled to display capabilities
- **Compression**: Optimized for bandwidth and quality
- **Caching**: Smart local storage for offline reliability
- **Encoding**: Hardware-accelerated decoding when available

## Network Requirements

### Internet Connectivity
- **Minimum Speed**: 5 Mbps download

- **Latency**: <100ms for responsive updates
- **Reliability**: 99%+ uptime for commercial use

### Firewall Configuration
